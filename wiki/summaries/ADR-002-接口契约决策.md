# ADR-002: 模块间接口契约决策

**项目**: 培训机构教务收费管理系统 (EDU)
**ADR 编号**: ADR-002
**决策日期**: 2026-06-24
**决策者**: A5-需求验证智能体
**状态**: ✅ 已决策
**关联文档**: ASD-EDU-V1 (§4 通信与集成约束) / MDS-EDU-V1 (§3 接口契约概要) / DTS-EDU-V1

---

## 1. 决策背景

ASD 已确定 RESTful 资源导向 API 风格和统一响应体，但以下细节尚未决策：

| 待决策项 | 背景 |
|---|---|
| OAS 文档范围 | 是否所有模块都需要完整的 OpenAPI 规范文档 |
| RESTful 资源路径约定 | 嵌套资源深度限制、批量操作路径规范 |
| 统一响应体细化 | 成功/业务异常/系统异常的分层响应格式 |
| 跨模块调用的接口契约 | Service 层跨模块调用是否需要显式接口声明 |
| 分页/排序/过滤规范 | 查询参数的统一命名和格式 |

---

## 2. 决策分析

### 决策 1: OAS 文档生成范围

| 方案 | 描述 | 优点 | 缺点 |
|---|---|---|---|
| A: 全量 | 所有 Controller 公开方法均生成 OpenAPI | 文档完整 | 内部接口（如管理后台）暴露 |
| B: 按需 | 仅对外暴露的 API（小程序端/第三方回调）生成 | 安全，不暴露内部接口 | 内部接口缺少文档 |
| **C: 分层** | 按 API 路径前缀区分：`/api/public/`（开放）、`/api/internal/`（内部），均生成文档但内部接口加 `@Hidden` 可配置 | 兼顾完整性和安全性 | 需维护路径前缀约定 |

**✅ 决策: 方案 C — 分层 OAS 文档**

| 路径前缀 | OAS 文档 | 访问控制 |
|---|---|---|
| `/api/v1/public/**` | ✅ 全量生成 | 无特殊限制 |
| `/api/v1/admin/**` | ✅ 生成但加 `x-internal: true` | 需管理员角色 |
| `/api/v1/callback/**` | ✅ 生成（支付回调） | IP 白名单 + 证书验证 |

### 决策 2: RESTful 资源路径约定

| 规则 | 约定 | 示例 |
|---|---|---|
| 资源嵌套深度 | ≤ 2 层 | `GET /api/v1/courses/{id}/classes`（✅）；`GET /api/v1/courses/{id}/classes/{cid}/students/{sid}`（❌ 超过 2 层） |
| 集合操作 | 复数资源名 | `GET /api/v1/students?campus_id=1` |
| 单资源操作 | `/{resource}/{id}` | `GET /api/v1/students/123` |
| 子资源 | `/{parent}/{pid}/{child}` | `GET /api/v1/orders/456/payments` |
| 批量操作 | `POST /{resource}/batch` | `POST /api/v1/students/batch-import` |
| 非 CRUD 动作 | 动词后缀 | `POST /api/v1/orders/456/cancel`, `POST /api/v1/attendance/789/check-in` |

### 决策 3: 统一响应体规范

```json
// 成功响应
{
  "code": 0,
  "message": "success",
  "data": { ... },
  "timestamp": "2026-06-24T10:00:00Z",
  "traceId": "a1b2c3d4"
}

// 业务异常（客户端错误）
{
  "code": 100101,
  "message": "手机号格式错误",
  "data": null,
  "timestamp": "2026-06-24T10:00:00Z",
  "traceId": "a1b2c3d4"
}

// 分页响应
{
  "code": 0,
  "message": "success",
  "data": {
    "content": [ ... ],
    "page": 0,
    "size": 20,
    "totalElements": 150,
    "totalPages": 8
  },
  "timestamp": "2026-06-24T10:00:00Z",
  "traceId": "a1b2c3d4"
}
```

| 字段 | 说明 |
|---|---|
| `code` | `0`=成功，`1xxxxx`=业务异常（6位模块错误码），`9xxxxx`=系统异常 |
| `message` | 人类可读的提示信息（可展示给前端用户） |
| `data` | 业务数据（分页时为含 `content/page/size/totalElements/totalPages` 的对象） |
| `timestamp` | ISO 8601 UTC 时间戳 |
| `traceId` | 全链路追踪 ID（MDC 注入） |

### 决策 4: 错误码编码规则

```
{模块2位}{类别2位}{序号2位}

模块: 01=CRM, 02=ENR, 03=ACA, 04=FIN, 05=RPT, 06=ADM, 00=CORE
类别: 01=参数校验, 02=业务规则, 03=权限, 04=外部调用, 05=数据
```

| 错误码 | 含义 |
|---|---|
| `0` | 成功 |
| `010101` | CRM-参数校验-手机号格式错误 |
| `020201` | ENR-业务规则-订单已支付不可修改 |
| `030301` | ACA-权限-非本校区教务不可操作 |
| `040401` | FIN-外部调用-支付网关超时 |
| `000501` | CORE-数据-数据库连接失败 |
| `999999` | 未知系统异常 |

### 决策 5: 跨模块 Service 调用契约

| 规则 | 说明 |
|---|---|
| **接口声明** | 每个模块对外暴露的 Service 必须定义 `interface`（如 `StudentService`），调用方依赖接口而非 `impl` |
| **包位置** | 对外接口放在模块根包（如 `edu.crm.StudentService`），实现放在 `service/impl/` |
| **返回值** | 禁止返回 Entity——跨模块调用返回 DTO（防止调用方绕过 Service 直接操作数据） |
| **审计** | 跨模块调用自动记录日志（通过 AOP 拦截所有 `*.service.*` 的 public 方法） |

---

## 3. 决策影响

| 影响维度 | 说明 |
|---|---|
| **前端开发** | 统一响应体 + 错误码规范使前端可以统一处理成功/异常/分页逻辑 |
| **测试** | OAS 文档可自动生成 Mock Server（Springdoc + Prism），前端可并行开发 |
| **运维** | `traceId` 使日志可全链路追踪（Nginx → Controller → Service → Repository → SQL） |
| **安全性** | 分层 OAS 避免内部管理接口暴露在公网文档中 |
