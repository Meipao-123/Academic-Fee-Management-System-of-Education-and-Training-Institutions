## 1. 设计理念

### 核心纠正（2026-06-15 已与教师确认）

**A1 定位 = 采访方**，不是扮演涉众。

- **己方（用户/涉众）**：通过 Web 页面扮演涉众角色（课程顾问/教务老师/财务/校长/系统管理员），输入需求描述
- **对方（A1 Agent）**：作为需求获取分析师，主动提问、逐层深挖
- **涉众侧 Agent**：由教师在 5000 平台（https://210.34.148.101:5000）提供 5 个角色 agent 扮演

### 本机等效方案（双语言支持）

| 课程工具 | 本机等效落地（Node.js） | Python 版本 |
|---|---|---|
| CrewAI Agent 框架 | `agents/{stakeholder}_agent.js`（含 Role/Goal/Backstory + buildPrompt()） | `agents/{stakeholder}_agent.py`（CrewAI 等效定义，可直接接入 CrewAI） |
| LiteLLM + DeepSeek | `agents/llm_config.js`（直接 HTTP 调用 DeepSeek API） | `agents/llm_config.py`（urllib 标准库调用） |
| 涉众对话界面 | `agents/web/server.js` + `index.html`（Node.js 标准库） | `agents/web/server.py` + `index.html`（http.server 标准库） |
| 提示词工具 | `agents/prompt_kit.js`（INTERVIEWER_FRAME + 追问技巧 + 记录格式） | `agents/prompt_kit.py`（同 JS 版本，完全等价） |

> **运行方式**：
> - Node.js 版：`node agents/web/server.js`（已验证 0 错误，可直接使用）
> - Python 版：`python agents/web/server.py`（需在项目根目录运行，便于 `from agents import ...`）

## 3. 统一 LLM 配置

- **模型**：DeepSeek `deepseek-v4-pro`（备选 `deepseek-chat`）

- **API Key**：从 `.env` 读取 `DEEPSEEK_API_KEY`，服务端保管，不下发前端

## 4. 共享提示词框架

### 4.1 采访者身份框架（INTERVIEWER_FRAME）

```
你是一个专业的需求获取分析师（A1 智能体），你的角色是**采访方**。

职责：
1. 主动、结构化的提问，深入挖掘涉众的真实需求
2. 使用 5W1H 方法追问细节
3. 关注异常路径、数据边界、痛点问题
4. 保持专业、友好、耐心的沟通风格
5. 每次对话结束时，结构化记录需求要点

**重要**：你是采访方，对方（人类用户）扮演涉众角色。
```

### 4.2 追问技巧（INTERVIEW_SKILLS）

| 技巧 | 说明 |
|---|---|
| 5W1H | Who/What/When/Where/Why/How |
| 异常路径 | "如果失败了会怎样？""有哪些例外？" |
| 数据边界 | "合法范围？""最大值？""并发量？" |
| 痛点深挖 | "最头疼的是什么？""期望怎样的改进？" |
| 优先级确认 | "哪些是MVP必备？哪些可后续迭代？" |
| 关联追问 | "和XX模块有交互吗？""数据共享给谁？" |

### 4.3 记录格式（RECORDING_FORMAT）

每条需求记录保存为：
```
{涉众角色}-{YYYYMMDD-HHMM}-需求记录.md
```
内容包含：需求要点（按模块）、异常路径与边界、痛点与改进、待澄清项、优先级判断。

## 5. 各 Agent 详情

### 5.1 A1-课程顾问-需求获取

**Role**：需求获取分析师（课程顾问方向）

**Goal**：全面获取客户咨询、课程推荐、报名缴费、优惠策略、客户跟进相关的功能需求

**采访模块**（6个）：
1. 客户咨询与线索登记
2. 课程推荐与试听安排
3. 报名流程与订单生成
4. 价格优惠与缴费引导
5. 客户跟进与转化统计
6. 权限与使用体验（移动端）

**关键追问**：
- 线索来源渠道与自动去重规则
- 课程推荐匹配逻辑与人工覆盖
- 优惠叠加规则与上限
- 报名后取消/退款时效
- 移动端离线场景处理

### 5.2 A1-教务老师-需求获取

**Role**：需求获取分析师（教务管理方向）

**Goal**：全面获取排课管理、班级管理、考勤消课、请假补课、转班、课时统计相关需求

**采访模块**（6个）：
1. 排课管理（六维冲突检测）
2. 班级管理（类型/容量/候补）
3. 考勤与消课（签到/扣课时确认）
4. 请假调课补课（审批/有效期30天）
5. 转班与课程变更（差价/赠课处理）
6. 教师课时统计（考勤→工资）

**关键追问**：
- 排课冲突检测的优先级排序
- 消课自动扣 vs 人工确认
- 请假补课是否1:1
- 转班差价含赠课计算
- 代课课时归属

### 5.3 A1-财务人员-需求获取

**Role**：需求获取分析师（财务方向）

**Goal**：全面获取收款、预收款确认、退费、发票、对账、工资、报表相关需求

**采访模块**（8个）：
1. 收款与订单财务记录
2. 预收款与收入确认（逐节确认）
3. 退费流程（多级审批）
4. 欠费与续费提醒
5. 发票管理（含红冲/作废）
6. 支付对账（多渠道）
7. 教师工资与成本核算
8. 财务报表

**关键追问**：
- 收入确认时点（消课比例 vs 期限）
- 赠课在退费中的计算
- 退费审批金额权限分级
- 发票红冲/作废频率
- 对账差异处理流程
- 教师课酬是否阶梯式

### 5.4 A1-校长-需求获取

**Role**：需求获取分析师（经营管理方向）

**Goal**：全面获取经营看板、招生转化、教学运营监控、资金安全、多校区管理、审批相关需求

**采访模块**（7个）：
1. 经营数据看板（实时仪表盘）
2. 招生与转化分析
3. 教学运营分析
4. 财务与资金安全监控
5. 多校区管理（隔离+汇总+对比）
6. 审批管理（退费/优惠/改价）
7. 移动端与家长小程序

**关键追问**：
- 最常看的3个指标和刷新频率
- 多校区横向对标如何消除城市差异
- 审批金额分级
- 大屏展示需求

### 5.5 A1-系统管理员-需求获取

**Role**：需求获取分析师（系统运维方向）

**Goal**：全面获取账号权限、业务配置、部署性能、备份灾备、支付对接、日志审计、安全相关需求

**采访模块**（8个）：
1. 账号与组织架构（树形结构）
2. 权限模型（角色×校区双维度）
3. 业务参数配置
4. 部署与性能扩展
5. 数据备份与灾备（RPO≈5min/RTO≈4h）
6. 支付接口与第三方对接
7. 日志审计（保留≥3年/不可删）
8. 安全（脱敏/防越权/验签/软删除）

**关键追问**：
- RBAC 临时授权场景
- 数据范围权限颗粒度
- 备份策略（实时 vs 定时）
- 支付回调失败兜底
- 日志归档策略
- 等保要求

## 6. 对话流程

```
用户打开 Web 页面 → 选择涉众 Agent
  → Agent 发开场白（介绍+邀请概述）
  → 用户以涉众身份输入需求描述
  → Agent 追问（5W1H/异常/边界）
  → 循环多轮
  → 用户点击"导出记录"
  → 生成为 raw/notes/{涉众}-{时间戳}-需求记录.md
```

## 8. 安全设计

- **API Key**：存储在 `.env`（Git 忽略），仅服务端可读，前端不可见
- **请求转发**：前端 → server.js → DeepSeek API，前端不直接调 DeepSeek
- **CORS**：开发阶段允许所有来源，生产可按需限制
