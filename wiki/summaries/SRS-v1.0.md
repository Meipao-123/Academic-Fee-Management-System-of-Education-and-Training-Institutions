## 1. 引言

### 1.1 目的
本文档定义"教育培训机构教务收费管理系统"的软件需求规格说明 (Software Requirements Specification, SRS)，作为后续软件设计、编码实现、测试验证和需求变更管理的基准。

本文档面向以下读者：
- **开发团队 (Reasonix)**: 基于本文档生成 v1 代码，并据此执行 v2 变更
- **A5 需求验证智能体**: 对照 raw/notes/ 原始记录进行交叉验证
- **CCB (变更控制委员会)**: 作为基线审批依据
- **教师/评审**: 期中考核 (第1-2周成果) 和期末考核 (第3-4周成果) 的核心评分项

### 1.2 范围
本系统覆盖**教育培训机构**的核心业务场景：
1. **销售咨询** — 线索管理、课程推荐、报名缴费、优惠策略、客户跟进
2. **教务管理** — 排课管理、班级管理、考勤消课、请假补课、转班、教师课时统计
3. **财务管理** — 收款、预收款/收入确认、退费、发票、支付对账、教师工资、财务报表
4. **经营管理** — 经营数据看板、招生分析、运营监控、资金安全、多校区管理、审批管理
5. **系统运维** — 账号/权限、业务配置、部署/备份、支付对接、日志审计、安全

## 2. 总体描述

### 2.1 产品视角
本系统是一个 **B/S 架构的 SaaS 教育管理系统**，部署于阿里云，服务约 500 注册用户 / 100 并发。系统通过 **Web 管理后台** 和 **家长端小程序** 两个入口提供服务，对接微信支付、支付宝支付、短信通道、电子发票平台等外部系统。

```
┌─────────────────────────────────────────────┐
│           教育培训机构教务收费管理系统          │
├─────────────────────────────────────────────┤
│  Web 管理后台  │  家长端小程序  │  开放 API     │
├─────────────────────────────────────────────┤
│  业务层: 销售 │ 教务 │ 财务 │ 经营 │ 系统       │
├─────────────────────────────────────────────┤
│  数据层: MySQL(业务) + Redis(缓存)             │
├─────────────────────────────────────────────┤
│  基础设施: 阿里云 ECS + 备份 + 日志审计         │
└─────────────────────────────────────────────┘
```

### 2.2 用户特征
| 角色 | 人数 | 技术水平 | 使用频率 |
|---|---|---|---|
| 课程顾问 | 5-10 | 中等 (日常使用办公软件) | 每天高频 (8h/天) |
| 教务老师 | 3-5 | 中等 | 每天中频 |
| 财务人员 | 1-2 | 较高 (财务系统经验) | 工作日中频 |
| 校长 | 1 | 中等 | 每天看板/审批 |
| 系统管理员 | 1 | 高 (IT 背景) | 按需 |
| 家长/学员 | 200-500 | 基础 (微信小程序) | 按需低频 |

### 2.3 约束
1. **技术栈**: Java + Spring Boot 四层架构 (Controller → Service → Repository → Entity)
2. **数据库**: MySQL 8.0 业务库 + Redis 缓存
3. **部署**: 阿里云 ECS，单应用单库起步，支持后续水平扩展
4. **支付**: 微信支付 + 支付宝 + 银行转账 + 现金，所有在线支付需验签
5. **安全合规**: 敏感数据脱敏 (手机号/身份证)、RBAC 双维度权限、日志保留 ≥3 年
6. **软删除**: 所有核心数据采用软删除 (逻辑删除)，物理删除延迟 ≥90 天
7. **AI 生成代码**: v1 代码由 Reasonix 按三层约束 (TLCD) + OAS 契约生成

### 2.4 假设与依赖
1. 机构已有营业执照和支付商户号
2. 第三方服务 (微信支付/支付宝/短信) 的 API 密钥由系统管理员配置
3. 家长小程序由单独的小程序项目承载，本系统提供后端 API
4. DeepSeek API (`deepseek-v4-pro`) 稳定可用，Token 费率在预算内
5. [TBD: 部署环境的具体规格 (CPU/内存/带宽等) 待确认]
